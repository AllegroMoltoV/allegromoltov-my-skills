# my-skills

Claude Code で使用する開発スキル集です。プロジェクト管理、テスト、ドキュメント作成における規約やベストプラクティスを定義しています。

## 背景

Claude Code を活用したソフトウェア開発では、コードの品質だけでなく、コミットメッセージの書き方やドキュメントの文体など、開発プロセス全体に一貫性が求められます。本リポジトリは、こうした規約をスキルファイルとして管理し、プロジェクト間で再利用できるようにしたものです。

## 内容物

### SKILLS/

開発ガイドラインを格納するディレクトリです。

| ファイル | 概要 |
|---|---|
| `PROJECT-SKILL.md` | Git の運用ルール、計画の承認フロー、Bash 実行時の注意点など、プロジェクト管理全般の規約 |
| `TEST-SKILL.md` | テスト失敗時の対処方針やカバレッジ目標など、テストに関する規約 |
| `DOC-SKILL.md` | 日本語技術文書の文体、表記ルール、構成方針に関するスタイルガイド |

### その他

| ファイル | 概要 |
|---|---|
| `INIT.md` | Claude Code の `/init` 命令で使用する初期指示 |
| `exclude` | `.git/info/exclude` の参照用コピー |

## 使い方

1. 対象プロジェクトの `.prompts/SKILLS/` に、必要なスキルファイルをコピーし、`.prompts/` に `INIT.md` をコピーします。
2. `git init` コマンドを入力後、`.git/info/exclude` の中身を `exclude` で置き換えます。
3. Claude Code に `.prompts/INIT.md` の内容を実行するよう依頼します。
